Часть №1. Как мне определить свой вариант?
Отлично, хороший вопрос. Для того, чтобы определить свой вариант, вам необходимо вписать вашу фамилию большими латинскими буквами в переменную SURNAME и запустить код, что ниже

public class PickVariant {

    private static final String SURNAME = "DRONOV";
    private static final int VARIANTS_COUNT = 4;

    public static void main(String[] args) {
        System.out.println(new PickVariant().hashCode());
    }

    @Override
    public int hashCode() {
        return Math.abs(SURNAME.hashCode()) % VARIANTS_COUNT + 1;
    }
}


Полученная цифра является вашим вариантом. Что с ней делать мы обсудим ниже.

Часть №2. А что надо сделать то?
А правильные вопросы задаете, давайте разбираться:
Вам необходимо реализовать одну из представленных структур данных (разумеется, соответствующую вашему только что полученному варианту), а также описать инвариант, пред- и постусловия.

Варианты:
LinkedDeque (дек на связном списке)
ArrayDeque (дек на массиве)
LinkedList (связный список)
ArrayList (список на массиве)

Для LinkedDeque и ArrayDeque необходимо реализовать интерфейс java.util.Deque.
Для LinkedList и ArrayList интферфейс java.util.List

Для успешной сдачи домашнего задания необходимо реализовать назначенную структуру данных в одной из двух версий:

Базовая версия:
реализовать все методы за исключением методов, работающих с итераторами (Iterator)
описать инвариант класса, пред- и постусловия всех методов (даже утилитарных)
методы toString(), hashCode()
тесты к реализованному классу
Усложненная версия:
весь функционал из Базовой версии
реализовать методы работающие с итераторами
динамическое расширение списков (только для ArrayList, ArrayDeque) в 2 стороны: увеличение при нехватке места и уменьшении при куче свободного места
использование Generic
каждый метод должен иметь адекватную асимптотику работы. Описать асимптотику работы каждого метода.

Будет большим плюсом помимо Базовой версии, сделать какую-то часть Усложненной версии.
